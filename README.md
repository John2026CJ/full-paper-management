# full-paper-management V3.1

全流程文献管理技能 —— 对文件夹中的文献文件执行七项标准化任务，实现从文件整合到 Markdown 转换的全自动文献库管理。

## V3.1 更新日志

| # | 变更 | 说明 |
|---|------|------|
| 1 | **主题词库上限** | 100 篇文献内最多 10 个主题词，超限时强制复用已有词或归入「未分类」 |
| 2 | **用户确认机制** | Task 5 按主题分类时，推断主题列表后需用户确认方可执行 |
| 3 | **MD 重试后缀** | 写后验证失败时使用 `_retry1` 后缀重试 |
| 4 | **排除逻辑增强** | Task 7 增量处理中增加前缀匹配排除 |

## 七项任务

| 任务 | 说明 | 操作类型 |
|------|------|----------|
| **任务一** | 文件整合 → `Combined` 文件夹 | 复制 |
| **任务二** | 文件重命名（`AuthorYYYY.ext` 格式） | 重命名 |
| **任务三** | 重复判别与冗余整合 → `Duplicates` | 移动 |
| **任务四** | 生成/更新 `LitsSum.xlsx`（6 列，含主题词） | 写入 |
| **任务五** | 双重分类（`ByYear` + `ByTopic`，**含用户确认**） | 复制 |
| **任务六** | Markdown 转换 + 三文件夹分类（`md_all`/`md_ByYear`/`md_ByTopic`） | 转换+复制 |
| **任务七** | 增量新增（多源发现、重命名、去重、更新） | 混合 |

## 文件结构

```
full-paper-management/
├── SKILL.md                     # 技能规范（7 项任务 + 9 条全局规则）
├── README.md                    # 本文件
├── README_EN.md                 # 英文说明
└── scripts/
    ├── metadata_utils.py         # 元数据提取、DOI 解析、APA 构建、主题词提取
    ├── file_ops.py               # 文件整合、重命名、去重、双重分类、MD 分类
    ├── litsum_manager.py         # LitsSum.xlsx 创建/读取/更新（6 列）
    └── md_converter.py           # PDF/DOCX/TXT → MD 转换 + 结构化模板
```

## LitsSum.xlsx 表结构

| 列 | 表头 | 说明 |
|---|---|---|
| A | 出版年份 | 四位数字，Unknown 排最后 |
| B | 文件名 | 去扩展名 |
| C | 完整引用(APA) | APA 7th 格式 |
| D | 原始文件夹 | 源头文件夹名 |
| E | 关键词 | 分号分隔，最多 5 个 |
| F | 主题词 | 1-2 个，V3.1 上限 10/100篇 |

## 全局规则

1. **先预览后执行** — 所有写操作前输出 dry-run 清单
2. **精细化日志** — 状态码标注（✅ ⏭ ⚠️ 📦 📁 🔄 ❌）
3. **文件安全** — 严禁删除原始文件
4. **稳健异常处理** — 单步/全局容错
5. **严格执行顺序** — 首次：1→6，增量：任务七
6. **备份恢复** — LitsSum.xlsx 操作前自动备份
7. **幂等一致性** — 重复执行不产生重复数据
8. **资源限制** — 内容解析前 3000 字符，DOI 超时 10s
9. **统一递增** — 冲突文件名 `_1` 起递增

## 依赖

```bash
pip install PyMuPDF           # PDF 文本提取（推荐）

pip install PyPDF2            # PDF 文本提取（备选）
pip install python-docx       # DOCX 文本提取
```

## 使用方式

加载 Skill 后，在对话中说「文献管理」或「整理文献」即可触发。按提示确认每个任务的 dry-run 预览后执行。
