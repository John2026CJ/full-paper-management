# 全流程文献管理 V2 (Full Paper Management)

对文件夹中的文献文件执行完整的七项标准化任务。所有操作遵循 **先预览（dry-run）后执行** 原则，**不删除任何原始文件**。

## 文件范围定义

- **文献类型**: .pdf, .docx, .doc, .txt, .epub, .mobi
- **处理范围**: 递归遍历当前文件夹及所有子文件夹
- **排除项**: 隐藏文件、系统文件、代码文件、图片、README、Excel、PPT 等非文献文件

## 依赖检查

执行任务前，确保以下 Python 库可用：

```bash
pip install PyPDF2 python-docx openpyxl requests
```

如果依赖缺失，先安装再执行后续步骤。

## 脚本工具

四个核心脚本位于 scripts/ 目录：

| 脚本 | 功能 |
|------|------|
| scripts/metadata_utils.py | 元数据提取（PDF/DOCX/TXT、DOI 解析、APA 构建、关键词提取、冲突降级） |
| scripts/file_ops.py | 文件操作（收集、重命名、去重、年代/主题分类、MD 分类、空文件检测） |
| scripts/litsum_manager.py | LitsSum.xlsx 管理（创建、读取、更新、备份、排序、年份映射） |
| scripts/md_converter.py | Markdown 转换（PDF/DOCX/TXT 转 MD、空文件检测） |

## 任务一：文件整合 (Collect)

将分散在各级子文件夹中的文献汇聚到统一目录。

### 执行步骤

1. 确定 **当前工作目录**（用户指定的文献根目录）。
2. 创建目标文件夹，命名规则：上级文件夹名称_Combined。
   - 如上级文件夹为 Papers -> 目标文件夹为 Papers_Combined。
3. 使用 scripts/file_ops.py collect 执行收集：
   ```bash
   python scripts/file_ops.py collect 源目录 [--dest 目标目录] [--dry-run]
   ```
4. 先以 --dry-run 执行预览，确认后去掉 --dry-run 正式执行。
5. 冲突处理：若目标已存在同名文件，自动添加 _数字 后缀（从 1 递增）。

### 输出要求

整合结果清单，格式如下：

```
原路径 -> 目标路径
```

## 任务二：文件重命名 (Rename)

将文献文件名标准化为 第一作者姓氏年份[后缀].扩展名 格式。

### 命名规则（V2 增强）

- **基础格式**: 第一作者姓氏 + 年份 + [区分后缀] + 扩展名
- **格式约束**: 姓氏与年份之间严禁出现空格、下划线或其他分隔符
- **默认规则**: 首选仅使用第一作者的姓氏（如 Smith2020.pdf）
- **冲突降级**: 当检测到同姓氏+同年份对应不同文献时，退化为"姓氏+名字全拼"形式
  - 触发条件: 检测到同姓氏、同年份且推断为不同作者
  - 示例: Zhang Yuan 与 Zhang Fei 均有 2020 年文献 -> ZhangYuan2020.pdf 和 ZhangFei2020.pdf
- **非英文姓名**: 中文姓名在冲突时使用拼音全名（如 WangXiaoming2023.pdf）
- **特殊字符**: 忽略姓名中的连字符、间隔号或撇号，直接拼接

### 信息提取优先级

按序尝试，取到即止：
1. **PDF 元数据**: Author, Title, CreationDate
2. **内容解析**: 提取文档前 2000 字符，匹配作者+年份
3. **文件名推断**: 从原名中提取年份和作者线索
4. **DOI 解析** (可选): 若文件含 DOI，通过 Crossref API 获取

### 执行步骤

```bash
python scripts/file_ops.py rename 目标目录 [--dry-run]
```

### 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 无法提取作者或年份 | 保留原文件名，记录到跳过清单 |
| 非英文作者 | 使用拼音姓氏（如 Zhang、Li）；冲突时用拼音全名 |
| 无发表年份 | 尝试 CreationDate；仍无则用 UnknownYear |
| 重名不同文献 | 年份后加小写字母: Smith2020a.pdf, Smith2020b.pdf |
| 同一文献多文件 | 年份后加下划线数字: Smith2020_1.pdf |
| 含特殊字符 | 转为下划线连接，不允许空格 |

### 输出要求

重命名映射表：

| 原文件名 | 新文件名 | 状态 | 备注 |
|----------|----------|------|------|
| report.pdf | Smith2020.pdf | 成功 | 从PDF元数据提取 |
| paper.docx | ZhangYuan2023.pdf | 成功 | 冲突降级为全名 |
| notes.txt | - | 跳过 | 无法提取作者信息 |

## 任务三：重复文献判别与整合 (Dedup)

检测并处理重复文献，每组保留最优版本，冗余文件移入 Duplicates 文件夹。

### 重复判定标准（满足任一即算重复）

1. **DOI 完全相同**
2. **作者+年份+标题前 40 字符完全一致**
3. **MD5 哈希相同**
4. **文件名编辑距离 <= 3** 且作者+年份一致

### 最优文件选择规则（优先级递减）

1. **信息完整性最高** - 量化打分（满分 10）：
   - 含摘要 +1，正文 >= 3页 +3，参考文献 +2，附录 +2，图表 +1，DOI +1
2. **文字清晰度最优**（页数代理）
3. **文件容量更小**
4. **格式优先**: .pdf > .docx > .doc > .txt > .epub > .mobi

### 执行步骤

```bash
# 1. 检测重复（预览）
python scripts/file_ops.py dedup 目标目录

# 2. 移动冗余文件
python scripts/file_ops.py dedup 目标目录 --move [--dry-run]
```

### 输出要求

| 组号 | 保留文件 | 冗余文件 | 选择理由 |
|------|----------|----------|----------|
| 1 | Smith2020.pdf | Smith2020_compressed.pdf | 得分 9/10, 页数更多 |

## 任务四：文献格式转换为 Markdown (MD Convert)

将 Combined 文件夹中的文献文件转换为 Markdown 格式，分类整理并检测空文件。

### Step 1：扫描待转换文件

1. 定位到 Combined 文件夹。
2. 递归扫描所有文献文件（包含各年份子文件夹）。
3. **严格排除** Duplicates 文件夹中的所有文件。
4. 输出待转换文件清单供用户确认。

### Step 2：执行格式转换

使用 scripts/md_converter.py 执行转换：

```bash
python scripts/md_converter.py convert Combined目录 [--dry-run]
```

- 转换能力判定：若环境具备 PDF/DOCX 文本层解析能力，直接转换。
- 不具备时，提示用户调用相应格式转换技能。
- 转换后的 .md 文件保存在原文件所在目录，命名为 原文件名.md。
- 不删除原文件，不覆盖已有 MD（冲突加 _数字 后缀）。

### Step 3：转换后处理

输出转换结果报告：

| 原文件名 | 状态 | 备注 |
|----------|------|------|
| Smith2020.pdf | 已转换 | 生成 Smith2020.md |
| Chen2024.pdf | 跳过 | 扫描版PDF，需OCR支持 |

### Step 4：分类 Markdown 文件

将转换成功的 .md 文件按年份分类到 _Com_md 文件夹：

```bash
python scripts/file_ops.py classify_md Combined目录 [--md-dir Com_md目录] [--year-map year_map.json] [--dry-run]
```

- 文件夹命名规则：上级文件夹名称_Com_md（如 Papers_Com_md）
- 按十年区间分类（同任务六），年份来源优先级：LitsSum 列 A -> 文件名推断 -> 文件元数据
- 无法获取年份放入 UnknownYear 子文件夹
- Duplicates 中的 .md 文件不参与分类

### Step 5：空文件检测与提醒

```bash
python scripts/md_converter.py check Com_md目录
```

判定标准：

| 情况 | 判定 | 处理 |
|------|------|------|
| 0 KB | 转换失败 | 记录并提醒用户 |
| < 1 KB | 高度疑似失败 | 记录并提醒用户 |
| >= 1 KB | 正常 | 不额外处理 |

检测到空文件或小文件时，输出提醒并询问用户是否删除（需确认后才删除）。

## 任务五：生成 / 更新 LitsSum 文献表 (LitsSum)

在 Combined 文件夹中创建或更新 LitsSum.xlsx。

### 表结构（V2 新增列 E）

| 列 | 表头 | 说明 |
|----|------|------|
| A | 出版年份 | 四位数字年份，无法提取则 Unknown |
| B | 文件名 | 去除扩展名后的纯文件名 |
| C | 完整引用(APA) | APA 格式文献引用 |
| D | 原始文件夹 | 文件最初来源文件夹名 |
| E | 关键词 | 最多 5 个关键词，以分号分隔 |

### 元数据提取优先级

1. **DOI 解析** - Crossref API（作者、年份、标题、期刊、卷号、页码、DOI、关键词）
2. **PDF 元数据** - Author, Title, CreationDate
3. **文件内容解析** - 前 3000 字符
4. **文件名推断**

### 关键词提取规则

从以下来源依次尝试提取最多 5 个关键词：
1. DOI 解析返回的 Keywords/Subject 字段
2. 文件内容前 3000 字符中频率最高的 3-5 个学术术语（排除停用词）
3. 标题中的核心术语
4. 均无法获取则留空

关键词之间用分号（;）分隔。

### APA 格式规则（V2 逐字段规范）

**英文文献模板：**
Author, A. A., & Author, B. B. (Year). Title of article. *Journal Name*, *Volume*(Issue), Page-Page. https://doi.org/xxx

**逐字段规则：**

| 字段 | 规则 |
|------|------|
| 作者 | 姓, 首字母缩写. 多作者逗号分隔，最后用 & 连接 |
| 年份 | (Year). 四位数字 |
| 标题 | 仅首字母大写（专有名词除外），句点结束 |
| 期刊名 | 斜体，每个主要单词首字母大写 |
| 卷号 | 斜体，紧跟期号 |
| 期号 | 括号内不斜体 |
| 页码 | 短横线连接 |
| DOI | 以 https://doi.org/ 开头 |

**中文文献模板：**
作者. (年份). 文章标题. 期刊名称, 卷号(期号), 页码.

**缺失字段处理：** 直接省略，不猜测。

**强制自检（每条 APA 生成后必须执行）：**
1. 作者格式是否为 姓, 首字母缩写. 或中文全名
2. 年份是否用括号包裹并以句点结束
3. 标题是否仅首字母大写（英文）
4. 期刊名是否斜体
5. 卷号斜体、期号不斜体
6. 页码格式是否为 起始页-结束页.
7. DOI 是否以 https://doi.org/ 开头
8. 是否存在多余标点

### 执行步骤

```bash
# 1. 备份已有 LitsSum（若存在）
python scripts/litsum_manager.py backup LitsSum路径

# 2. 提取元数据（批量，含关键词和 APA）
python scripts/metadata_utils.py --batch --apa 目标目录 -o metadata_batch.json

# 3. 更新 LitsSum
python scripts/litsum_manager.py update LitsSum路径 --entries metadata_batch.json [--dry-run]

# 4. 排序
python scripts/litsum_manager.py sort LitsSum路径
```

### 去重规则

- 判定标准：DOI 相同 / 作者+年份+标题前 40 字符一致 / MD5 相同
- 新文献：追加新行
- 重复且新数据更完整：更新列 C（保留更完整版本），列 B 保留首次出现的文件名
- 重复且旧数据更完整：跳过
- **关键词列（E）**：新提取的关键词与已有数据对比，保留关键词数量更多的一方；数量相同则保留新提取的版本

## 任务六：文献分类整理（按年份 / 按主题）

### Step 0：确认分类模式

主动询问用户选择分类模式：
- **模式 A：按年份分类**（默认十年区间）
- **模式 B：按主题分类**（需提供主题列表）

禁止自行决定分类方式，必须先询问。

### 模式 A：按年份分类

分类区间：

| 年份范围 | 文件夹名 |
|----------|----------|
| <= 1989 | 1900-1989 |
| 1990-1999 | 1990-1999 |
| 2000-2009 | 2000-2009 |
| 2010-2019 | 2010-2019 |
| 2020-2029 | 2020-2029 |
| 无法识别 | UnknownYear |

```bash
# 从 LitsSum 构建年份映射
python scripts/litsum_manager.py yearmap LitsSum路径 -o year_map.json

# 执行分类
python scripts/file_ops.py classify 目标目录 --mode year --year-map year_map.json [--dry-run]
```

### 模式 B：按主题分类

#### 1. 获取主题列表（优先级链）

| 优先级 | 来源 |
|--------|------|
| 1 | 用户直接提供 |
| 2 | 已有 LitsSum.xlsx 中列 C 推断 |
| 3 | 文件名 + 文件元数据推断 |
| 4 | 无信息时请用户手动提供 |

推断结果展示后必须等待用户确认。

#### 2. 主题文件夹命名

格式：序号_主题名称（如 01_机器学习, 02_自然语言处理）

#### 3. 未归类文献

无法归入任何主题的文献放入 UnClassified 文件夹。

```bash
# 执行主题分类
python scripts/file_ops.py classify 目标目录 --mode topic --topics topics.json [--topic-map topic_map.json] [--dry-run]
```

### 分类依据来源

| 优先级 | 来源 | 适用场景 |
|--------|------|----------|
| 1 | LitsSum.xlsx 列 A（出版年份） | 按年份 |
| 2 | LitsSum.xlsx 列 C（完整引用） | 按主题 |
| 3 | 文件名推断 | 两种模式备选 |
| 4 | 文件元数据 | 年份缺失时补充 |

### 注意事项

- Duplicates 文件夹不参与分类
- 支持重复执行（幂等）
- 模式切换时需确认是否重新分类已有文件

## 任务七：新增文献的增量处理 (Incremental)

当用户在工作目录中新增了包含文献的文件夹后，执行增量入库流程。

### 总体流程: Step 0 -> 1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 7

### Step 0：发现新增源文件夹

1. 扫描工作目录，识别所有非 Combined、非 Duplicates、非系统文件夹。
2. 输出新增源文件夹清单。
3. 若未发现，输出"未检测到新增源文件夹，无需处理"，任务结束。

### Step 1：重命名新增文献

对新增源文件夹中的文件执行任务二的命名规则（含冲突降级）。**原地重命名**（不复制）。

```bash
python scripts/file_ops.py rename 新增源文件夹 [--dry-run]
```

记录原始文件夹映射关系（用于后续填充 LitsSum 列 D）。

### Step 2：预整合

将重命名后的文件**复制**到 Combined 根目录。冲突时加 _数字 后缀。保留源文件夹原始文件不删除。

### Step 3：识别新增文献

判定标准（同时满足）：
- 位于 Combined 根目录（未分类）
- 文件名不在 LitsSum 已有记录的列 B 中
- 不与已分类到年代文件夹中的文件同名

### Step 4：判别新增文献中的重复

对新增文件之间、新增与库存之间执行任务三的重复检测。冗余文件移入 Duplicates。

### Step 5：更新 LitsSum

对新增保留文件执行任务五的元数据提取和表更新。
- 列 D 填写新增源文件夹名（非 Combined）
- 列 E 填写提取的关键词
- 去重规则：新数据更完整则更新列 C，列 B 保留首次出现值

### Step 6：分类新增文献

对新增保留文件执行任务六的分类规则。文件移动到对应的年代/主题文件夹。

### Step 7：排序与汇总

对 LitsSum 表按列 A 升序排列（Unknown 排最后）。输出统计汇总表：

| 任务 | 新增源文件夹 | 新增文件 | 更新记录 | 跳过 | 已移动 |
|------|------------|----------|----------|------|--------|
| 重命名 | 2 | 8 | - | 1 | - |
| 预整合 | - | 8 | - | - | - |
| 重复整合 | - | - | - | 1 | 2 |
| 更新LitsSum | - | 6 | 0 | - | - |
| 年代分类 | - | 6 | - | - | 6 |

### 注意事项

- 不影响已有数据：LitsSum 中已有行不被修改（除非发现更完整版本）
- 不重新分类已有文件：仅处理新增文献
- 幂等性：重复执行不会产生重复数据
- 重命名先于整合：所有文件先在源文件夹完成标准化重命名，再复制到主库

## 全局规则

1. **先预览，后执行**: 每项任务先用 --dry-run 预览操作清单，用户确认后再正式执行。
2. **不删除原始文件**: 所有操作为复制（任务一、七 Step 2）、重命名（任务二、七 Step 1）、移动（任务三、四、六、七 Step 4/6），绝不删除。
3. **输出日志**: 区分"成功""跳过""冲突已处理""已移动""已更新"五种状态。
4. **幂等性**: 支持重复执行，不会重复创建文件夹或移动已分类文件。
5. **备份机制**: 操作 LitsSum.xlsx 前自动备份为 LitsSum_backup_YYYYMMDD_HHmmss.xlsx。
6. **错误处理**: 单个文件处理失败不中断整体流程，记录原因后跳过。
7. **异常处理**: 任何任务失败不中断整体流程，记录原因后继续下一任务。
