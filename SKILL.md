# 全流程文献管理技能 V3.1

对文件夹中的所有文献文件完成以下七项标准化任务。**在执行任何修改操作之前，必须先输出 dry-run 预览清单并等待用户确认。**

---

## 文件范围定义

- **文献文件类型**：`.pdf`、`.docx`、`.doc`、`.txt`、`.epub`、`.mobi`
- **处理范围**：递归遍历当前文件夹及所有子文件夹
- **排除项**：隐藏文件、系统文件、代码文件、图片、README、以及明显非文献的文件

---

## 全局要求（9 条强制执行规范）

> 以下条款适用于所有七个任务，必须严格遵守，不可偏离。

### 1. 先预览后执行（Dry-Run 机制）
在执行任何写文件、移动、重命名、复制操作之前，必须先输出操作预览清单，并等待用户确认。
- 预览内容必须包含：即将创建/修改的文件路径、具体操作类型、涉及的任务三（重复分组与移动计划）的详细映射。
- 执行时机：仅在用户回复"确认"、"执行"或类似肯定指令后方可动手。

### 2. 精细化日志输出
所有输出必须明确标注状态码：✅ 成功 / ⏭ 跳过 / ⚠️ 冲突已处理 / 📦 已复制 / 📁 已移动 / 🔄 已更新 / ❌ 失败。

### 3. 文件安全与操作类型
严禁删除任何原始文献文件（PDF/DOCX/TXT 等）。
- 复制（Copy）：任务一、任务五、任务六、任务七 Step 2/6
- 重命名（Rename）：任务二、任务七 Step 1（原地重命名）
- 移动（Move）：任务三、任务六 Step 4-6、任务七 Step 4
- 冲突处理：若目标路径存在同名文件，必须添加递增数字后缀（如 `file_1.pdf`），严禁覆盖。

### 4. 稳健的异常处理
- 单步容错：任一文件处理失败，必须记录原因并标记为 ❌ 失败，然后立即处理下一个文件。
- 全局容错：任一任务失败，必须记录错误详情，继续执行后续任务。

### 5. 严格的任务执行顺序
首次运行（全量处理）：
1. 任务一：文件整合
2. 任务二：文件重命名
3. 任务三：重复文献判别与冗余整合
4. 任务四：生成 LitsSum.xlsx
5. 任务五：文献分类（ByYear / ByTopic）
6. 任务六：Markdown 转换与分类

增量运行（任务七）：当用户表达"新增文献"或"更新库"时触发。假设任务一至六已执行过。
1. Step 0：发现新增源文件夹
2. Step 1：重命名新增文献
3. Step 2：预整合至 Combined
4. Step 3：识别新增文献
5. Step 4：判别新增文献中的重复文件
6. Step 5：更新 LitsSum 表
7. Step 6：分类新增文献（含 MD 转换）

### 6. 备份与恢复机制
- 备份对象：仅针对 LitsSum.xlsx
- 备份时机：在执行任何可能修改 Excel 内容的操作前（任务四、任务七）
- 备份命名规则：`LitsSum_backup_YYYYMMDD_HHmmss.xlsx`

### 7. 幂等性与一致性
- 幂等性：重复执行同一脚本不会导致数据重复、文件夹嵌套混乱或 Excel 出现重复行
- 命名一致性：文件夹命名严格遵循 `上级文件夹名称_XXX` 格式
- 数据一致性：
  - Excel 列 D（原始文件夹）记录源头文件夹名
  - Excel 列 F（主题词）与 ByTopic 文件夹名称严格对应
  - 执行完毕后输出最终统计汇总表，核对各环节文件数量一致性

### 8. 资源限制与性能
- 文件内容解析仅读取前 3000 字符
- DOI 解析、外部查询设超时 10 秒，超时即降级处理
- MD 文件容量检测（0 KB 或 < 1 KB 视为失败，提醒 OCR）

### 9. 统一递增规则
冲突文件名后缀从 `_1` 开始递增，直到找到可用文件名。

## 任务一：文件整合

1. 在当前工作目录下创建文件夹 `上级文件夹名称_Combined`
2. 若已存在，先清空再重建（或询问用户）
3. 将所有文献文件**复制**到新文件夹，**保留原始文件不删除**
4. 同名冲突时添加递增数字：`文件名_数字.扩展名`（如 `Smith2020_1.pdf`），数字从 1 开始
5. 复制完成后输出整合结果清单

---

## 任务二：文件重命名

对 `上级文件夹名称_Combined` 中的文件逐一重命名。

### 重命名规则
- 基础格式：`<第一作者标识><发表年份>[区分后缀].<扩展名>`（如 `Smith2020.pdf`）
- 严禁空格、下划线分隔
- 默认：仅使用第一作者姓氏
- 冲突降级：同姓氏+同年份+不同文献 → 退化为"姓氏+名字"全拼（如 `ZhangYuan2020.pdf`）
- 非英文姓名：中文姓名使用拼音全名（如 `WangXiaoming2023.pdf`）
- 特殊字符：忽略连字符、间隔号、撇号，直接拼接

### 信息提取优先级
1. PDF 元数据（Author、Title、CreationDate）
2. 文件内容解析（前 2000 字符）
3. 文件名推断
4. DOI 解析（仅当前 3 项失败时）

### 特殊情况
| 情况 | 处理方式 |
|---|---|
| 无法提取作者或年份 | 保留原文件名不变，记录跳过清单 |
| 非英文作者名 | 优先拼音全名，无法转写则保留原文 |
| 无发表年份 | CreationDate 年份 → `UnknownYear` |
| 重名冲突（不同文献） | 年份后加小写字母：`Smith2020a.pdf` |
| 同一文献多个文件 | 年份后加下划线数字：`Smith2020_1.pdf` |

## 任务三：重复文献判别与冗余文件整合

### Step 1：判别同一文献的重复文件
按以下优先级链逐一比对，满足任一即视为同一篇：
1. DOI 完全相同
2. 作者+年份+标题前 40 字符完全一致
3. 文件内容 MD5 哈希相同
4. 文件名高度相似（编辑距离 ≤ 3）且作者+年份一致

### Step 2：每组选择最优保留文件
量化打分（满分 10 分）：摘要 +1、正文≥3页 +3、参考文献 +2、附录 +2、图表 +1、DOI +1。
得分最高者保留；得分相同则：文字清晰度 → 文件容量较小 → 格式优先（`.pdf > .docx > .doc > .txt`）。

### Step 3：冗余文件移动到 `Duplicates`
- 在 `Combined` 中创建 `Duplicates` 文件夹
- 未被选中的冗余文件移动到 `Duplicates`
- 保留文件不动，继续留在 Combined

## 任务四：生成 / 更新 LitsSum.xlsx

### 表结构（6 列）
| 列 | 表头 | 说明 |
|---|---|---|
| A | 出版年份 | 四位数字；Unknown 排最后 |
| B | 文件名 | 去除扩展名 |
| C | 完整引用(APA) | APA 格式文献引用 |
| D | 原始文件夹 | 文件最初所在文件夹名称 |
| E | 关键词 | 分号分隔，最多 5 个 |
| F | 主题词 | 1-2 个，用于文件夹命名 |

### Step 1：读取已有数据
若 `LitsSum.xlsx` 已存在 → 先备份 → 读取全部数据建立判定键。

### Step 2：提取文献元数据
优先级：DOI 解析 → PDF 元数据 → 文件内容解析（前 3000 字符）→ 文件名推断。

关键词提取：最多 5 个，分号分隔。

### Step 3：按 APA 格式生成"完整引用"
- 英文：`Author, A. A. (Year). Title. Journal Name, Volume(Issue), Page-Page. https://doi.org/xxx`
- 中文：`作者. (年份). 文章标题. 期刊名称, 卷号(期号), 页码.`
- 每次生成后必须逐条自检 8 项（最多重试 3 次）

### Step 4：去重与更新判定
判定标准：DOI 相同 / 作者+年份+标题前40字符 / MD5 相同。
- 新文献 → 追加新行
- 重复且新数据更完整 → 更新列 C，列 B 保留首次文件名
- 重复且信息完整度相同或更少 → 跳过

### Step 5：写入 Excel
- Sheet 名：`LitsSum`；表头第 1 行
- 按列 A（出版年份）升序排列，Unknown 排最后

### Step 6：汇总主题词（V3.1 新增上限规则）

> **V3.1 新增：主题词库上限 — 100 篇文献内的主题词总数控制在 10 个之内。**

1. **提取范围**：应用于所有文献行。空白行判定：列 F 为空/仅空格/值为 Unknown/N/A/—。
2. **提取规则**：
   - 数量限制：严格 1-2 个词
   - **主题词库上限**：100 篇文献内 ≤ 10 个主题词。每次生成新词前先检查词库数量，已达 10 个则必须从已有词中选最语义匹配的复用，严禁新增。若已有词均不匹配，归入 `未分类`，报告中说明原因。
   - 连接规则：英文 `and`，中文 `和`
   - 实词首字母大写，缩写全大写，`and` 小写
   - 总长度 ≤ 30 字符
3. **新旧主题词对齐**：
   - 建立历史词库（扫描列 F 非空值）
   - 完全匹配 → 复用历史词
   - 语义重合但格式不同 → 优先历史写法
   - 仅部分概念重合 → 沿用历史词，不凑第二个词
   - 完全无匹配 → 生成新词（受上限约束）
4. **输出主题词汇总报告**。

## 任务五：文献分类整理（按年份 / 按主题）

> 本任务对文献进行**双重分类**：按年份 → `上级文件夹名称_ByYear`，按主题 → `上级文件夹名称_ByTopic`。两种分类同时执行。

### Step 1A：按年份分类（十年区间）
| 年份范围 | 文件夹名称 |
|---|---|
| ≤ 1989 | `1900-1989` |
| 1990-1999 | `1990-1999` |
| 2000-2009 | `2000-2009` |
| 2010-2019 | `2010-2019` |
| 2020-2029 | `2020-2029` |
| 无法识别 | `UnknownYear` |

年份来源优先级：LitsSum 列 A → 文件名推断 → 文件元数据。

### Step 1B：按主题分类（V3.1 新增用户确认机制）

**获取主题列表优先级：**
1. LitsSum.xlsx 列 F（主题词）非空值 → 收集去重
2. 用户直接提供 → 严格使用
3. 列 F 部分为空 → 推断缺失主题词后补充
4. 无 LitsSum 但有已分类年份文件夹 → 读取文件名+元数据推断

**V3.1 用户确认机制：**
- 展示推断结果后，**必须等待用户确认或修改**后再执行分类
- 用户可输入自定义主题列表，AI 必须严格遵循
- 用户可输入"按默认"或"默认"或留空 → 使用 AI 推断的主题列表
- 文件量极少（≤5 个）→ 直接询问用户，不推断
- 无法提取有效关键词 → 归入 `未分类`，报告中说明原因

**主题匹配规则：**
- 一篇文献一个主题（取列 F 第一个主题词或 and/和 连接的两个词视为一个主题）
- 列 F 为空 → APA 引用推断 → 语义相似度匹配 → `未分类`

**执行：** 从 Combined 复制到 ByTopic/ByYear 对应子文件夹。Duplicates 不参与分类。

## 任务六：文献格式转换为 Markdown

### Step 1：创建目标文件夹结构
```
上级文件夹名称_md/
├── 上级文件夹名称_md_all/       # 所有 .md 文件
├── 上级文件夹名称_md_ByYear/    # 按年份分类的 .md
└── 上级文件夹名称_md_ByTopic/   # 按主题分类的 .md
```

### Step 2：扫描待转换文件
从 Combined 递归扫描，严格排除 Duplicates 文件夹。

### Step 3：执行格式转换并写出 .md 文件

**结构化内容模板：**
```markdown
# [文献标题]

**作者：** [作者名]
**年份：** [年份]
**来源：** [期刊名/会议名]

---

[正文内容]
```

**写后强制验证（V3.1 细化）：**
- 检查目标 .md 文件是否存在且大小 > 0
- 若未成功写出，必须**重试一次**（重试时加 `_retry1` 后缀避免覆盖）
- 重试仍失败则记录原因并跳过
- 禁止在未写出文件的情况下进入下一步

### Step 4-6：复制到三个 MD 文件夹
- Step 4：复制所有 .md → `md_all`
- Step 5：按十年区间复制 → `md_ByYear`
- Step 6：按主题词复制 → `md_ByTopic`
- 所有操作均为复制，不移动、不删除

### Step 7：空文件检测
- 0 KB → 转换失败
- < 1 KB → 高度疑似失败
- 检测到空/小文件后询问用户是否删除（Y 确认 / N 保留）

## 任务七：新增文献的增量处理

> 触发条件：用户表达"新增文献"或"更新库"。假设任务一至六已执行过。

### Step 0：发现新增源文件夹
扫描 Combined 父目录，识别所有新增源文件夹。排除项（V3.1 增强）：
- 文件夹名称**完全匹配**或**以 `上级文件夹名称_` 为前缀**的，一律排除
- 排除清单：Combined、Duplicates、ByYear、ByTopic、md、md_all、md_ByYear、md_ByTopic、系统文件夹

### Step 1：重命名新增源文件夹中的文献
规则与任务二完全一致。操作类型：原地重命名。记录原始文件夹映射关系。

### Step 2：预整合新增文献
将重命名后的文件复制到 Combined 根目录。保留源文件不删除。

### Step 3：识别新增文献
基于标准化文件名判定。不在 LitsSum、不在已分类年份文件夹 → 新增。

### Step 4：判别新增文献中的重复文件
新增文件互相比对 + 与 Combined 已有文件比对。冗余文件移动到 Duplicates。

### Step 5：更新 LitsSum 表
完全沿用任务四流程：元数据提取 → APA 构建 → 主题词提取（含上限规则）→ 去重写入。

### Step 6：分类新增文献
- 按年份分类到 ByYear
- 按主题分类到 ByTopic
- MD 转换后复制到 md_all / md_ByYear / md_ByTopic

### Step 7：排序与汇总
LitsSum 表按列 A 升序排列，输出最终统计汇总表。
